## Phase Goal

**As a** job seeker, **I want to** have job leads automatically scraped, filtered, matched to my best resume, and applied to via email with a tailored cover letter and Telegram notification, **so that** I never miss an eligible opportunity.

<objective>
Build the Gmail ingestion vertical slice: OAuth2 client that authenticates with Gmail API, historyId-based polling that detects new LinkedIn job alert emails, and two FastAPI endpoints (/poll-gmail, /fetch-email-body) that n8n will call on an hourly schedule. The historyId checkpoint is persisted in the AgentConfig SQLite table.

Purpose: Gmail ingestion (INGEST-01) is the primary lead source. The system must detect new LinkedIn job alert digest emails within an hour of arrival and make their plain-text bodies available for AI extraction.
Output: Working Gmail polling endpoints testable via curl; OAuth token acquisition script; historyId persistence in SQLite; unit tests covering key behaviors.
</objective>

    Create src/ingestion/gmail_client.py with three functions per RESEARCH.md Pattern 3 and PATTERNS.md gmail_client.py section.

    Function get_gmail_service(token_path: str) -> Resource: Load credentials from token_path via Credentials.from_authorized_user_file(token_path). If creds.expired and creds.refresh_token, call creds.refresh(Request()) and write the refreshed token back to token_path. Return build("gmail", "v1", credentials=creds). Wrap the refresh call with tenacity @retry(stop=stop_after_attempt(3), wait=wait_exponential(min=1, max=10)) for transient network errors. Use structlog.get_logger() for logging.

    Function poll_gmail_since(service, start_history_id: str | None, sender_filter: str) -> tuple[list[str], str]: Per RESEARCH.md Pattern 3. If start_history_id is None (first run), call messages.list(userId="me", q=f"from:{sender_filter}", maxResults=1) to get baseline historyId from the most recent message. Return ([], historyId). If start_history_id is provided, call history.list(userId="me", startHistoryId=start_history_id, historyTypes=["messageAdded"]) with pagination. Collect all message IDs from messagesAdded events. For each message ID, fetch metadata headers (From) and filter to only messages where sender_filter appears in the From header. Return (matching_ids, latest_history_id). Wrap history.list() in try/except for HttpError with status 404 -- on 404, log warning "gmail_history_id_expired", call recursively with start_history_id=None to reset baseline per RESEARCH.md Pitfall 1.

    Function fetch_message_body(service, message_id: str) -> dict: Call messages.get(userId="me", id=message_id, format="full"). Extract the plain text body from the MIME payload. For multipart messages, iterate parts looking for mimeType "text/plain". Base64url-decode the body data. Also extract Subject and From headers. Return {"body_text": decoded_text, "subject": subject, "sender": sender}.

    Imports: google.oauth2.credentials.Credentials, google.auth.transport.requests.Request, googleapiclient.discovery.build, googleapiclient.errors.HttpError, base64, structlog, tenacity.
