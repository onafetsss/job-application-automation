## Phase Goal

**As a** job seeker, **I want to** have job leads automatically scraped, filtered, matched to my best resume, and applied to via email with a tailored cover letter and Telegram notification, **so that** I never miss an eligible opportunity.

<objective>
Build the AI preparation and application state management vertical slice: resume text extraction from .docx/.pdf files, profile config loading, Claude Haiku-powered resume selection endpoint, and application lifecycle endpoints (/write-application, /mark-submitted) that transition job states and create Application records.

Purpose: AI-01 (resume selection), AI-02/AI-03 support (profile config for prompts), and APPLY-02 (application state transitions) are the core intelligence layer. The /select-resume endpoint provides n8n with the best-fit resume name + text for cover letter generation. The /write-application and /mark-submitted endpoints manage the job lifecycle from QUEUED through SUBMITTED.
Output: Working resume selection via LLM, application state management endpoints, profile config loaded at startup.
</objective>
