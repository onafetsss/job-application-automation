## Phase Goal

**As a** job seeker, **I want to** have job leads automatically scraped, filtered, matched to my best resume, and applied to via email with a tailored cover letter and Telegram notification, **so that** I never miss an eligible opportunity.

<objective>
Build the AI preparation and application state management vertical slice: resume text extraction from .docx/.pdf files, profile config loading, Claude Haiku-powered resume selection endpoint, screening answer generation endpoint (AI-03 per D-20), a GET /profile endpoint that serializes profile config for n8n (D-19), and application lifecycle endpoints (/write-application, /mark-submitted) that transition job states and create Application records.

Purpose: AI-01 (resume selection), AI-02 (profile config for cover letter prompts), AI-03 (screening answer generation), and APPLY-02 (application state transitions) are the core intelligence layer. The /select-resume endpoint provides n8n with the best-fit resume name + text. The /generate-screening-answers endpoint generates answers from screening questions + profile + JD via Haiku. The GET /profile endpoint exposes profile.yaml as JSON so n8n can inject it into the Claude Sonnet cover letter prompt (per D-19) instead of using a static block. The /write-application and /mark-submitted endpoints manage the job lifecycle from QUEUED through SUBMITTED.
Output: Working resume selection via LLM, screening answer generation via LLM, profile API endpoint, application state management endpoints, profile config loaded at startup.
</objective>
