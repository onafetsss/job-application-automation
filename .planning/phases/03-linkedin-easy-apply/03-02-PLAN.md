<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LinkedIn site → browser session | untrusted remote DOM controls what fields/pages are rendered |
| job.url (from DB) → page.goto | URL originates from filtered/ingested DB rows, not raw user input |
| profile/resume data → form fields | applicant PII (phone, email, resume) typed into LinkedIn forms |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Denial of Service | modal navigation loop | mitigate | Bounded loop with explicit Submit/Review/Next ARIA detection; raise on "no button found" rather than infinite-loop (Pitfall 3); challenge check before any interaction halts on /checkpoint/ |
| T-03-04 | Information Disclosure | resume file path / PII in logs | mitigate | structlog logs job_id and event names only — never the resume bytes or full profile PII; resume path resolved from RESUMES_DIR env |
| T-03-05 | Spoofing/Detection | bot fingerprint | mitigate | headless="virtual" + humanize=True + persistent_context (never headless=True); fingerprint smoke-test deferred to Plan 04 human checkpoint |
| T-03-06 | Tampering (injection) | screening answer generation from job description | mitigate | job_description truncated to 1500 chars in prompt; answers are returned to LinkedIn form only, not executed; Haiku output parsed as JSON, not eval'd |
| T-03-07 | Elevation/Availability | LinkedIn account ban | mitigate | challenge detection raises ChallengeDetected and stops immediately (D-06, no auto-retry); rate cap enforced in Plan 04 |
</threat_model>
