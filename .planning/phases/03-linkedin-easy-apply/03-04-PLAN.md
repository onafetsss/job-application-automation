## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| human → LinkedIn login | manual credential entry during session save (no credentials in code/JSON) |
| n8n → FastAPI | scheduled POST carrying job_id with fastApiKey header |
| FastAPI/Camoufox → LinkedIn | live automated session subject to bot detection + rate limits |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-12 | Information Disclosure | LinkedIn credentials / session cookies | mitigate | Credentials entered manually only; never in workflow JSON or code; session persisted to `data/linkedin_profile/` which is git-ignored (V3 session management) |
| T-03-13 | Availability | LinkedIn account ban from over-submission | mitigate | n8n daily-cap gate (LINKEDIN_DAILY_CAP, default 17) + time window + randomized 8-25min delay (D-09); challenge alert halts run (D-06); VPS-IP only (D-07) |
| T-03-14 | Spoofing/Detection | bot fingerprint exposure | mitigate | Blocking human checkpoint validates bot.sannysoft.com passes before any autonomous run (success criterion 2); headless="virtual"+humanize from Plan 02 |
| T-03-15 | Tampering | n8n workflow JSON pointing at wrong host | mitigate | verify gate asserts no `localhost` and all URLs are `http://job-app-api:8000` |
| T-03-16 | Denial of Service (hang) | challenge page on a live run | mitigate | challenge → 503 → n8n error output → Telegram alert + terminate; checkpoint step 5 verifies no silent hang (OPS-01/NOTIF-02) |
</threat_model>
