## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Spoofing | POST /apply/linkedin-easy-apply | mitigate | `dependencies=[Depends(verify_api_key)]` on the state-changing POST, matching all existing routes; GET counters read-only and unauthenticated per existing convention |
| T-03-09 | Tampering (SSRF) | job.url passed to page.goto | mitigate | URL is read from the DB Job row (already ingested + filtered), never from the n8n payload — payload carries only job_id which is validated against the DB |
| T-03-10 | Repudiation | submission outcome | mitigate | write_audit records APPLYING / SUBMITTED / SKIPPED events with job_id + reason for every transition (OPS-03) |
| T-03-11 | Denial of Service | unbounded LinkedIn submissions | mitigate | daily-linkedin-count is the durable cap source consumed by the n8n gate (Plan 04); challenge → 503 halts the run |
</threat_model>
