    Create `docker/supervisord.conf`:

    This is the process manager config for the single browser+API container. Processes start in priority order. Xvfb must be up before fluxbox and x11vnc start. The app (uvicorn) starts last so DISPLAY=:1 is ready.

    The x11vnc command uses `-rfbauth /root/.vnc/passwd` — the VNC password file is created during image build (see Dockerfile.browser below). The `-forever` flag keeps x11vnc running after the first VNC client disconnects. The `-shared` flag allows both Camoufox automation and Stefano's noVNC session to use the same display simultaneously (Camoufox is already paused at reCAPTCHA when Stefano connects).

    The noVNC websockify launcher path is `/usr/bin/websockify` on Debian stable. The noVNC web assets are at `/usr/share/novnc` on Debian stable (Pitfall A6 — verify path on build; if missing, check `dpkg -L novnc | grep index.html`).

    ```ini
    [supervisord]
    nodaemon=true
    logfile=/var/log/supervisor/supervisord.log
    pidfile=/var/run/supervisord.pid

    [program:xvfb]
    command=/usr/bin/Xvfb :1 -screen 0 1920x1080x24
    autorestart=true
    priority=10
    stdout_logfile=/var/log/supervisor/xvfb.log
    stderr_logfile=/var/log/supervisor/xvfb.log

    [program:fluxbox]
    command=/usr/bin/fluxbox
    environment=DISPLAY=":1"
    autorestart=true
    priority=20
    stdout_logfile=/var/log/supervisor/fluxbox.log
    stderr_logfile=/var/log/supervisor/fluxbox.log

    [program:x11vnc]
    command=/usr/bin/x11vnc -display :1 -rfbauth /root/.vnc/passwd -forever -shared -rfbport 5900 -noxdamage
    autorestart=true
    priority=30
    stdout_logfile=/var/log/supervisor/x11vnc.log
    stderr_logfile=/var/log/supervisor/x11vnc.log

    [program:novnc]
    command=/usr/bin/websockify --web=/usr/share/novnc 6080 localhost:5900
    autorestart=true
    priority=40
    stdout_logfile=/var/log/supervisor/novnc.log
    stderr_logfile=/var/log/supervisor/novnc.log

    [program:app]
    command=uv run uvicorn src.api.app:app --host 0.0.0.0 --port 8000
    directory=/app
    environment=DISPLAY=":1",CAMOUFOX_DISPLAY_MODE="xvfb"
    autorestart=true
    priority=50
    stdout_logfile=/var/log/supervisor/app.log
    stderr_logfile=/var/log/supervisor/app.log
    ```

    Create `docker/Dockerfile.browser`:

    This extends `Dockerfile.api` by adding the display stack packages and supervisor config. The VNC password is baked in at build time using `x11vnc -storepasswd`. The password value is passed as a build arg `VNC_PASSWD` (required at build time — see compose in Plan 03). The Camoufox fetch layer is inherited from `Dockerfile.api` (do NOT re-fetch here — use a multi-stage or extend the api image directly).

    Key points:
    - Base: `python:3.11-slim` (same as Dockerfile.api — replicate the api setup then extend)
    - Add apt packages: `xvfb x11vnc novnc websockify fluxbox supervisor` (xvfb is already in Dockerfile.api but include for completeness — apt idempotent)
    - Create VNC password file during build: `RUN mkdir -p /root/.vnc && x11vnc -storepasswd "${VNC_PASSWD}" /root/.vnc/passwd && chmod 600 /root/.vnc/passwd`
    - Copy supervisord.conf into `/etc/supervisor/conf.d/app.conf`
    - EXPOSE 8000 (API) and 6080 (noVNC/websockify) — do NOT expose 5900 (VNC raw, not needed externally)
    - CMD is `/usr/bin/supervisord -c /etc/supervisor/supervisord.conf` — supervisor manages everything

    ```dockerfile
    FROM python:3.11-slim
    WORKDIR /app

    # Python deps
    COPY pyproject.toml uv.lock ./
    RUN pip install uv && uv sync --no-dev

    # Display stack + supervisor (Debian stable apt packages — all in 'main', curated)
    # xvfb: virtual X framebuffer; x11vnc: exposes display over VNC; novnc+websockify: HTML5 VNC client;
    # fluxbox: window manager (modal focus); supervisor: multi-process container manager
    RUN apt-get update && apt-get install -y --no-install-recommends \
        xvfb \
        x11vnc \
        novnc \
        websockify \
        fluxbox \
        supervisor \
        && rm -rf /var/lib/apt/lists/*

    # Camoufox Firefox binary (~100MB — cached as its own layer)
    RUN python -m camoufox fetch

    # VNC password baked at build time (passed as --build-arg VNC_PASSWD=<secret>)
    # This avoids embedding the password in the image layer history as plaintext.
    ARG VNC_PASSWD
    RUN mkdir -p /root/.vnc && \
        x11vnc -storepasswd "${VNC_PASSWD}" /root/.vnc/passwd && \
        chmod 600 /root/.vnc/passwd

    # Application source
    COPY src/ src/
    COPY main.py .

    # Supervisor config (manages Xvfb, fluxbox, x11vnc, noVNC, uvicorn)
    COPY docker/supervisord.conf /etc/supervisor/conf.d/app.conf

    # Log dir for supervisor
    RUN mkdir -p /var/log/supervisor

    EXPOSE 8000
    EXPOSE 6080

    CMD ["/usr/bin/supervisord", "-c", "/etc/supervisor/supervisord.conf"]
    ```

    Note for executor: if `dpkg -L novnc` on the VPS shows noVNC web assets at a path other than `/usr/share/novnc`, update the `--web=` argument in supervisord.conf to match. The Dockerfile.browser build will reveal this at `docker build` time on the VPS.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build arg VNC_PASSWD → image layer | Build arg is not stored in final image ENV but is in layer history of the build step. Mitigated by using `RUN x11vnc -storepasswd` (writes to file) rather than ENV — the password is not readable from `docker inspect`. |
| CAMOUFOX_DISPLAY_MODE in supervisor env | Set in supervisord.conf as a container-internal env var; not a secret. Controls headless mode only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-I | Information Disclosure | VNC_PASSWD build arg | mitigate | Use `x11vnc -storepasswd` to write to `/root/.vnc/passwd` (binary hash, not plaintext). Do not set as ENV. Pass as `--build-arg` at build time only. |
| T-05-02-T | Tampering | .google_credentials.json / .google_token.json | mitigate | Added to .gitignore this plan. `git check-ignore` gate in verify confirms exclusion before any git push. |

| T-05-SC | Tampering | apt packages installed in Dockerfile.browser | accept | All six packages are Debian `main`, curated, 10-20yr provenance. Package Legitimacy Audit in RESEARCH.md confirms Approved for all. No PyPI/npm installs. |
