  <name>Task 1: Run read-only VPS discovery commands and capture output</name>

  SSH into the Hostinger VPS and run the following commands IN ORDER. Copy-paste the full output of each command block into a reply. Do NOT modify any files or run `docker compose up/down/restart` — this is read-only reconnaissance.

  <how-to-verify>
    Run all command blocks below on the VPS and paste back the output.

    ---
    BLOCK 1 — OS + resources:
    ```
    cat /etc/os-release
    uname -a
    free -h
    nproc
    df -hT
    ```

    BLOCK 2 — Docker landscape:
    ```
    docker --version
    docker compose version
    docker ps -a --format "table {{.Names}}\t{{.Image}}\t{{.Status}}\t{{.Ports}}"
    docker network ls
    ```

    BLOCK 3 — Proxy network inspection (replace `traefik-proxy` with the actual network name from BLOCK 2 if different):
    ```
    docker network inspect traefik-proxy 2>/dev/null || echo "NETWORK_NOT_FOUND: traefik-proxy"
    ```

    BLOCK 4 — Identify the reverse proxy container and its config:
    ```
    docker inspect $(docker ps -q) --format '{{.Name}} {{.Config.Image}}' 2>/dev/null | grep -iE 'traefik|caddy|nginx|proxy'
    ss -tlnp | grep -E ':80\b|:443\b'
    ```

    BLOCK 5 — n8n container labels (reveals exact router/certresolver naming to mirror):
    Replace `<n8n_container>` with the n8n container name from BLOCK 2 output:
    ```
    docker inspect <n8n_container> --format '{{json .Config.Labels}}' | python3 -c "import sys,json; [print(k,'=',v) for k,v in json.load(sys.stdin).items() if 'traefik' in k.lower()]"
    ```

    BLOCK 6 — Traefik container args (reveals certresolver name):
    Replace `<traefik_container>` with the Traefik container name from BLOCK 4 output:
    ```
    docker inspect <traefik_container> --format '{{json .Args}}' 2>/dev/null | python3 -c "import sys,json; [print(a) for a in json.load(sys.stdin)]"
    ```

    BLOCK 7 — Existing domains (avoid collisions, confirms domain Stefano controls):
    Replace `<n8n_container>` with the actual name:
    ```
    docker inspect <n8n_container> --format '{{json .Config.Labels}}' | python3 -c "import sys,json; [print(v) for k,v in json.load(sys.stdin).items() if 'Host(' in str(v)]"
    ```

    BLOCK 8 — Data directory filesystem type (SQLite WAL safety check):
    ```
    df -T /root /opt 2>/dev/null
    findmnt -T /root 2>/dev/null
    ```

    BLOCK 9 — Any existing domain configured for this VPS (DNS check):
    ```
    hostname -f 2>/dev/null || hostname
    cat /etc/hosts | grep -v '^#' | grep -v '^$'
    ```

    After pasting output, answer these questions:
    1. What subdomain should the noVNC endpoint use? (e.g. `recaptcha.yourdomain.com`) — needed for the DNS record and compose label. If you have no domain, say so and Tailscale Funnel will be the fallback.
    2. Where on the VPS should the agent code live? (default: `/opt/job-agent`) — this is the deployment root.
    3. What is your VPS SSH user (root or another user)?
  </how-to-verify>
