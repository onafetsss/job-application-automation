<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → recaptcha.runlucid.co | Exposes a LIVE, logged-in LinkedIn browser. Must be auth-gated + HTTPS. |
| VPS git → GitHub | Was a leaked PAT-in-URL; replaced with a read-only SSH deploy key. |
| Edits to /opt/stack | Shared compose with the user's production n8n — a bad edit could take n8n down. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-S | Spoofing | noVNC endpoint | mitigate | Caddy basic_auth (bcrypt hash) + HTTPS; strong password; no anonymous access to the logged-in browser |
| T-05-03-I | Info Disclosure | Leaked GitHub PAT in remote URL | mitigate | Replace with SSH deploy key (Task 3); user revokes the PAT after, zero-impact |
| T-05-03-D | Denial of Service | Shared /opt/stack compose | mitigate | Back up compose+Caddyfile first; `docker compose config` validation before apply; rebuild ONLY job-app-api+caddy; documented rollback to bf6d040 |
| T-05-03-T | Tampering | Secrets transfer | mitigate | rsync/scp over SSH; secrets gitignored (not in the repo pull); profile dir on host ext4 |
| T-05-03-SC | Supply chain | apt/pip installs | accept | Image deps already vetted in 05-02; no new package sources here |
</threat_model>
