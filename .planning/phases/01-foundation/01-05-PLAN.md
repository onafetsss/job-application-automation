<objective>
Close VERIFICATION gap 3 (truth 9 FAILED) and code review finding CR-02: `check_eligibility()` silently bypasses the `blocked_phrases` JD scan when a lead has `location=None`. A job scraped without a location field but with "must be authorized to work in the US" in its JD currently passes the filter — directly defeating the primary protection against US-auth-required jobs for a Philippines-based user.

Move the `blocked_phrases` JD scan (rule 4a) outside the `if location is not None:` guard so it runs unconditionally on `jd_lower`. The allowlist check (rule 4b) remains inside the guard because it has no meaningful behavior without a location string.

Purpose: Restores FILTER-02 correctness for the case where a scraper omits the location field. Critical for the user's filter protection.

Output: Patched `src/filter/eligibility.py` + new regression test in `tests/unit/test_eligibility.py`. TDD: failing test first (RED commit), then fix (GREEN commit).
</objective>
