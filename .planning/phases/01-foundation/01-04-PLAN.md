<objective>
Close VERIFICATION gap 1 and gap 2 (same root cause): dry-run mode silently passes within-batch duplicate URLs because `is_duplicate()` only checks the DB and no Job rows are written in dry-run.

Add an in-memory `seen_hashes: set[str]` accumulator to `main.run()` and check it before the DB call. The first occurrence of a url_hash is added to the set; the second occurrence is intercepted, emits a `DEDUP_SKIP` audit event, and prints a "DEDUP_SKIP" indicator line in dry-run terminal output.

Purpose: Restores SC-2 (duplicate appears once) and SC-3 (dry-run validates full pipeline) end-to-end in dry-run mode, so the dry-run output the user inspects accurately reflects what live mode would do.

Output: Patched `main.py` + new regression test in `tests/integration/test_dry_run_pipeline.py`. Run via TDD: failing test first (RED commit), then fix (GREEN commit).
</objective>
